# CyberSim — Симулятор защиты личных данных

> Образовательная платформа по кибербезопасности от **Банк Центр-Инвест**
> Разработана в рамках хакатона **UMIRHack**

## О проекте

Пользователи банков ежедневно сталкиваются с фишингом, вишингом и социальной инженерией — и теряют деньги, потому что не умеют распознавать атаки.

**CyberSim** — интерактивный симулятор, который формирует устойчивые поведенческие навыки через геймификацию. Пользователь учится на практике: попадает в реальные сценарии атак, ошибается, получает разбор от ИИ-наставника и запоминает правильную модель поведения.

**Ключевые особенности:**
- 9 игровых режимов, охватывающих разные типы кибератак
- ИИ-генерация уникальных сценариев через Groq (llama-3.3-70b)
- Система геймификации: HP, очки, лиги, достижения, сертификат
- Персональный разбор ошибок от ИИ-наставника
- Таблица лидеров в реальном времени

## Быстрый запуск

### Требования

- Docker и Docker Compose
- Бесплатный Groq API ключ: https://console.groq.com

### Установка и запуск

```bash
# 1. Клонировать репозиторий
git clone https://github.com/3xfap2/cyber-simulator.git
cd cyber-simulator

# 2. Создать файл переменных окружения
cp .env.example .env
# Открыть .env и заполнить GROQ_API_KEY и SECRET_KEY

# 3. Запустить все сервисы
docker-compose up --build

# Сайт доступен на http://localhost:3000
# API документация на http://localhost:8000/docs
```

### Запуск без Docker (разработка)

```bash
# Backend
cd backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# Frontend (в отдельном терминале)
cd frontend
npm install
npm run dev
# Открыть http://localhost:3000
```

## Переменные окружения

Создайте `.env` файл на основе `.env.example`:

| Переменная | Описание | Пример |
|---|---|---|
| `POSTGRES_DB` | Имя базы данных | `cybersim` |
| `POSTGRES_USER` | Пользователь БД | `cybersim` |
| `POSTGRES_PASSWORD` | Пароль БД | `strongpassword` |
| `DATABASE_URL` | Строка подключения к PostgreSQL | `postgresql://cybersim:pass@postgres:5432/cybersim` |
| `SECRET_KEY` | Секрет для JWT (мин. 32 символа) | `your-secret-key-32-chars-minimum` |
| `GROQ_API_KEY` | API ключ Groq для ИИ-функций | `gsk_...` |
| `NEXT_PUBLIC_API_URL` | URL бэкенда для фронта | `http://localhost:3000` |

## Как пользоваться

### 1. Регистрация и вход
Перейдите на сайт → введите имя пользователя, email и пароль → нажмите «Создать аккаунт».

### 2. Рабочий стол
После входа открывается рабочий стол с вашей статистикой (HP, очки, лига) и доступом ко всем режимам.

### 3. Основные сценарии (рекомендуем начать здесь)
- Нажмите **«Начать обучение»** → выберите уровень сложности
- Читайте письма, звонки, сообщения — определяйте угрозу
- Выбирайте правильное действие из 4 вариантов
- После ответа ИИ объясняет ошибку или хвалит за правильный выбор

### 4. Пример сценария (фишинг)
```
От: support@sberbank-security.ru
Тема: Ваш аккаунт будет заблокирован через 24 часа

Уважаемый клиент, подтвердите личность по ссылке:
http://sberbank-secure-login.ru/verify

Варианты:
A) Перейти по ссылке и ввести данные        ← НЕВЕРНО (фишинг)
B) Позвонить в банк по номеру на карте      ← ВЕРНО
C) Ответить на письмо                        ← НЕВЕРНО
D) Проигнорировать и удалить                ← ВЕРНО (тоже принято)
```

### 5. ИИ-режим
В играх с кнопкой **«Режим ИИ»** — каждый раз генерируются новые уникальные сценарии. Ни одного повторения.

### 6. Сертификат
После прохождения 50%+ сценариев с правильными ответами — перейдите в профиль → «Получить сертификат». Сертификат содержит QR-код для проверки подлинности.

## API документация

После запуска доступна интерактивная документация Swagger:
- Локально: http://localhost:8000/docs
- Продакшн: https://cyber-simulator-production.up.railway.app/docs

### Основные эндпоинты

```
POST /api/auth/register     — Регистрация
POST /api/auth/login        — Вход
GET  /api/auth/me           — Текущий пользователь

GET  /api/scenarios/levels  — Список уровней
GET  /api/scenarios/{id}    — Сценарий по ID
POST /api/progress/answer   — Отправить ответ

GET  /api/leaderboard       — Таблица лидеров
POST /api/certificates/generate — Получить сертификат
GET  /api/certificates/verify/{id} — Проверить сертификат
```
